## Image Optimization Tool 화면설계서

### 1. 전체 레이아웃 (Overall Layout)

애플리케이션은 **상단 제어 영역(2행)**, **중앙 리스트 콘텐츠 영역**, **하단 상태 표시줄(1행)**로 구성됩니다.

```
+--------------------------------------------------------------------------------------+
| A-1. 폴더 제어 행 (원본/대상 폴더 선택, 리로드, 경로, 초기화, 시작/취소)             |
+--------------------------------------------------------------------------------------+
| A-2. 보기 제어 행 (카드 높이 슬라이더, 병렬 처리 슬라이더, 상태 필터 버튼)           |
+--------------------------------------------------------------------------------------+
| B. 컨텐츠 리스트 패널                                                                |
| +----------------------------------------------------------------------------------+ |
| | [원본 경로 + 용량] [상태칩/결과] [대상 용량 + 경로]                              | |
| | [원본 미리보기 + 메타데이터] [중앙 결과정보] [대상 미리보기 + 메타데이터]        | |
| +----------------------------------------------------------------------------------+ |
| | ... (가상 스크롤 기반 반복)                                                      | |
+--------------------------------------------------------------------------------------+
| C. 하단 상태 표시줄 (1행 텍스트 + 배경 진행 애니메이션)                              |
+--------------------------------------------------------------------------------------+
```

### 2. 컴포넌트 상세 설명

#### A. 상단 제어 영역 (Top Control Area)

#### A-1. 폴더 제어 행

- **원본 폴더 컨트롤**
  - [ `원본 폴더 선택...` ] 버튼
  - [ `새로고침` ] 버튼
  - 선택 경로 표시 텍스트 (`div`, 클릭 시 클립보드 복사)
  - [ `초기화` ] 버튼
- **대상 폴더 컨트롤**
  - [ `대상 폴더 선택...` ] 버튼
  - [ `새로고침` ] 버튼
  - 선택 경로 표시 텍스트 (`div`, 클릭 시 클립보드 복사)
  - [ `초기화` ] 버튼
- **마이그레이션 실행 컨트롤 (오른쪽 고정)**
  - [ `마이그레이션 시작` ] 버튼
  - [ `취소` ] 버튼
  - 상태:
    - 시작 버튼: 원본/대상 폴더가 모두 선택되어야 활성화
    - 진행 중: 시작 버튼 비활성 + 텍스트 `처리 중...`

#### A-2. 보기 제어 행

- **카드 높이 슬라이더**
  - 범위: `180 ~ 2200`
  - 현재값 표시: `NNNpx`
- **병렬 처리 슬라이더**
  - 범위: CPU 코어 기반 동적 계산 (`min/max/default`)
  - 현재값 표시: `선택 병렬도 (코어 수)`
  - 마이그레이션 시작 시 백엔드 동시 처리 제한값으로 전달
- **상태 필터 토글 버튼 (On/Off)**
  - `대기`, `스킵`, `성공`, `실패`
  - 선택 상태만 리스트에 표시

#### B. 컨텐츠 리스트 패널 (Single Virtualized List)

- **리스트 특성**
  - 단일 리스트 + 카드 반복 구조
  - 가상 스크롤 적용 (대량 파일 대응)
  - 필터 적용 후 결과 개수/전체 개수 표시
- **카드 구조 (좌/중/우 3열)**
  - **좌측(원본)**
    - 상단: `원본 상대경로`(클릭 복사) + 우측 끝 `원본 용량`
    - 본문: 원본 이미지 미리보기
    - 우하단 플로팅 메타데이터: `DPI`, `해상도`, `컬러`
  - **중앙(상태)**
    - 상태칩: `대기`, `최적화 완료`, `스킵(복사)`, `실패`
    - 결과 텍스트:
      - 성공: `원본용량 -> 대상용량` (+ 감소율)
      - 실패: 에러 메시지
      - 대기: `마이그레이션 전`
  - **우측(대상)**
    - 상단: 좌측 끝 `대상 용량` + `대상 상대경로`(클릭 복사)
    - 본문: 동일 상대경로 대상 파일 미리보기
    - 파일이 없으면 안내 문구 표시
    - 우하단 플로팅 메타데이터: `DPI`, `해상도`, `컬러`
- **경로 복사 UX**
  - 상단 폴더 경로 및 카드 경로 클릭 시 클립보드 복사
  - 화면 중앙 토스트: `클립보드에 복사되었습니다`

#### B-1. 이미지 단독 뷰어 팝업

- 카드 이미지 클릭 시 모달 오버레이로 단독 뷰어 오픈
- 기능:
  - 확대/축소 버튼 (`-`, `+`)
  - 배율 표시/리셋 버튼 (`100%` 기준)
  - 마우스 휠 줌
  - 더블클릭 리셋
  - `ESC` 또는 바깥 영역 클릭 시 닫기

#### C. 하단 상태 표시줄 (Status Bar)

- **1행 통합 텍스트 표시**
  - 상태 메시지
  - 진행 수치 (`processed/total`, 성공/실패)
  - 원본/대상 파일 수 및 용량 비교
  - 용량 증감량 + 증감율(%)
- **배경 진행 애니메이션**
  - 진행률에 따라 좌 -> 우로 상태바 배경이 차오름

### 3. 마이그레이션 처리 정책

- 대상: `jpg`, `jpeg`, `png`, `tif`, `tiff`
- 규칙:
  - DPI 정보가 둘 다 있고 300 초과일 때만 `slimg resize --scale ... --output ...` 실행
  - DPI 없음 또는 300 이하이면 파일 복사(스킵)
- 취소:
  - 취소 플래그 감지 시 진행 중 작업 중단
  - 실행 중 `slimg` 프로세스는 강제 종료 처리

### 4. 성능/응답성 설계

- 리스트 가상화 렌더링
- 미리보기/메타데이터 지연 로딩 + 큐 기반 동시성 제한
- 스크롤 중 가시 영역 우선 로딩
- 진행 이벤트/결과 업데이트는 배치 반영(스로틀/플러시)로 UI 응답성 유지
