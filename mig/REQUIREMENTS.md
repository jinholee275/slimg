## Image Optimization Tool 개발 요구사항 정의서

### 1. 개요

- **프로젝트 명**: Image Optimization Tool
- **개발 목표**: 스캐너로 생성된 고해상도(300 DPI 초과) 이미지 파일들을 300 DPI로 일괄 변환하여 디스크 저장 공간을 최적화하고, 이미지 자산을 표준화하는 데스크톱 애플리케이션을 개발합니다.
- **추진 배경**: 다수의 스캔 이미지가 필요 이상으로 높은 해상도를 차지하여 저장 공간 비효율을 초래하고 있습니다. 이를 해결하기 위해 지정된 폴더 내 이미지들을 자동으로 분석하고 최적화하는 도구가 필요합니다.

---

### 2. 주요 기능 요구사항

- **1. 이미지 파일 처리**
    - 지원 대상 파일 형식은 `.jpg`, `.png`, `.tif` (TIFF)으로 합니다.
    - 향후 다른 이미지 형식을 추가할 수 있도록 확장 가능한 구조로 설계합니다.

- **2. DPI 기반 최적화**
    - 이미지의 메타데이터를 읽어 DPI(Dots Per Inch) 값을 확인합니다.
    - 300 DPI를 초과하는 이미지는 300 DPI로 변환하여 새로운 파일로 저장합니다.
    - 300 DPI 이하의 이미지는 원본 그대로 대상 폴더로 복사합니다.

- **3. 폴더 설정**
    - 사용자는 '원본 폴더(Source)'와 '대상 폴더(Destination)'를 직접 지정할 수 있어야 합니다.

- **4. 폴더 구조 유지**
    - 마이그레이션 작업 시, '원본 폴더'의 하위 폴더 구조를 '대상 폴더'에 그대로 유지해야 합니다.
    - 예: `원본폴더/부서A/문서1.jpg` -> `대상폴더/부서A/문서1.jpg`

- **5. 병렬 처리**
    - 대량의 이미지를 신속하게 처리하기 위해, 여러 이미지 파일을 동시에 변환하는 병렬(Parallel) 처리 기능을 구현해야 합니다.

---

### 3. 사용자 인터페이스 (UI/UX) 요구사항

- **1. 기본 레이아웃**
    - 화면을 좌/우 2개의 스플릿 뷰(Split View)로 구성합니다.

- **2. 원본 이미지 뷰어 (좌측)**
    - '원본 폴더' 내의 이미지 목록을 표시합니다.
    - 사용자가 목록에서 특정 이미지를 선택하면 해당 이미지의 미리보기를 보여줍니다.

- **3. 최적화된 이미지 뷰어 (우측)**
    - '대상 폴더' 내의 이미지 목록을 표시합니다.
    - 좌측 뷰어에서 원본 이미지를 선택하면, 우측 뷰어에서는 해당 이미지의 최적화된 버전(또는 복사본)을 자동으로 선택하여 미리보기를 보여줍니다.

- **4. 메타데이터 표시**
    - 좌측과 우측 뷰어에서 각각 선택된 이미지의 상세 메타데이터를 확인할 수 있어야 합니다.
    - 표시되어야 할 메타데이터 항목은 다음과 같습니다.
        - 파일 경로 (Full Path)
        - 파일 크기 (File Size, e.g., MB, KB)
        - DPI
        - 해상도 (Resolution, e.g., 3508x2480)
        - 컬러 모드 (Color Mode, e.g., RGB, CMYK, Grayscale)

---

### 4. 비기능적 및 시스템 요구사항

- **1. 처리 도구**
    - 이미지 DPI 변환의 핵심 로직은 오픈소스 도구인 `slimg` ([https://github.com/clroot/slimg](https://github.com/clroot/slimg))를 사용해야 합니다.

- **2. 성능**
    - 시스템의 멀티코어 프로세서를 효율적으로 활용하여 병렬 처리를 수행함으로써 전체 작업 시간을 최소화해야 합니다.

- **3. 플랫폼**
    - Tauri와 Vue.js (또는 유사 프론트엔드 프레임워크)를 기반으로 한 크로스플랫폼 데스크톱 애플리케이션으로 개발합니다.
